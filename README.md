# Mg-Ca-Zn Ternary Alloy ML Surrogate Model

A machine learning surrogate that replaces expensive CALPHAD thermodynamic calculations in a Mg-Ca-Zn ternary alloy diffusion simulation. Given alloy composition and temperature, the surrogate predicts thermodynamic and kinetic properties needed by a PDE-based multi-component diffusion solver.

---

## Problem Statement

Diffusion simulations of Mg-Ca-Zn alloys require thermodynamic properties at every spatial node and every time step. Computing these via CALPHAD is accurate but prohibitively slow when called millions of times per simulation. This project trains a fast ML surrogate to replace those calls with negligible loss in accuracy.

The surrogate learns a static mapping:

```
f(T, X_Ca, X_Zn) → (μ_Ca, μ_Mg, μ_Zn, Dv_Ca, Dv_Mg, Dv_Zn, Vm)
```

No temporal architecture is needed — each prediction is an independent thermodynamic state lookup.

---

## Inputs and Outputs

**Inputs (3)**

| Symbol | Description | Units | Range |
|--------|-------------|-------|-------|
| T | Temperature | K | 323 – 973 |
| X_Ca | Mole fraction of Ca | — | 1×10⁻⁵ – 0.20 |
| X_Zn | Mole fraction of Zn | — | 1×10⁻⁵ – 0.30 |

X_Mg is implicit: `X_Mg = 1 - X_Ca - X_Zn` (always ≥ 0.50 in this dataset).

**Outputs (7)**

| Symbol | Description | Units | Scale |
|--------|-------------|-------|-------|
| μ_Ca | Chemical potential of Ca | J/mol | ~−10⁴ |
| μ_Mg | Chemical potential of Mg | J/mol | ~−10⁴ |
| μ_Zn | Chemical potential of Zn | J/mol | ~−10⁴ |
| Dv_Ca | Vacancy diffusion coefficient of Ca | m²/s | ~10⁻²³ |
| Dv_Mg | Vacancy diffusion coefficient of Mg | m²/s | ~10⁻²⁵ |
| Dv_Zn | Vacancy diffusion coefficient of Zn | m²/s | ~10⁻²⁵ |
| Vm | Molar volume | m³/mol | ~10⁻⁵ |

---

## Dataset

- **75,000 data points** on a perfect full grid: 30 (T) × 50 (X_Ca) × 50 (X_Zn)
- All inputs are **linearly and uniformly spaced**
- Compositional simplex constraint satisfied throughout: X_Ca + X_Zn + X_Mg = 1
- Data generated from CALPHAD thermodynamic database calculations

## Setup

**1. Clone the repository**
```bash
git clone https://github.com/yourusername/GP1.git
cd GP1
```

**2. Create a virtual environment and install dependencies**
```bash
python -m venv venv
source venv/bin/activate        # on Windows: venv\Scripts\activate
pip install -r requirements.txt
```

**3. Add raw data files**

Place the original CALPHAD output files in `data/raw/`:
```
data/raw/input_data.txt     # columns: T, X_Ca, X_Zn (comma-separated)
data/raw/output_data.txt    # columns: mu_Ca, mu_Mg, mu_Zn, Dv_Ca, Dv_Mg, Dv_Zn, Vm (whitespace-separated)
```

## Preprocessing Design Decisions

**Log-transform diffusion coefficients**
Dv values span ~23 orders of magnitude and follow Arrhenius behaviour. The model predicts `log10(Dv)` and back-transforms to raw scale at inference. Predicting on the raw scale would make the loss function numerically unstable.

**No log-transform on inputs**
All three inputs (T, X_Ca, X_Zn) are linearly and uniformly spaced. StandardScaler is sufficient — no input log-transform needed.

**Scaler fitted on training data only**
`scaling_parameters.json` stores two sets of scaler parameters: full-dataset (for reference) and train-only (for use in model training). Always use the train-only parameters to avoid data leakage.

**Simplex constraint**
The input space is not a simple hypercube. The constraint `X_Ca + X_Zn ≤ 1` (equivalently `X_Mg ≥ 0`) is enforced at the data level and validated in `input_bounds_analysis.py`.

## Ensemble Model Optimization Results

**1. Data Splitting & Leakage Prevention**
Implemented a strict 80/20 `train_test_split`. The initial $R^2 = 1.0$ was identified as an in-sample overfitting artifact. The new evaluation is strictly on the held-out test set.

**2. Model Pruning & Pipeline Scaling**
- Wrapped base learners in a `Pipeline` with `StandardScaler` to handle the massive scale differences between outputs (e.g., $\mu$ vs $D_v$).
- Pruned the Random Forest (`max_depth=15`, `min_samples_leaf=5`) to prevent overfitting.
- Upgraded the Meta-Learner to `RidgeCV` for automatic regularization tuning.

**3. Performance & The "Ridge Anomaly"**
- **Final Stacking Ensemble Score**: Mean $R^2 = 0.9999$ on the unseen test set.
- **Note on Base Ridge Model**: The base Ridge model scored poorly (e.g., -555). This is expected physics: diffusivity ($D_v$) scales exponentially with temperature, rendering a strictly linear model incapable. However, the Stacking Meta-Learner successfully identified this noise, assigning near-zero weight to the Ridge predictions and relying on the tree-based models to achieve a highly robust and accurate final output.
