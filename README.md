# Mg-Ca-Zn Ternary Alloy ML Surrogate Model

A machine learning surrogate that replaces expensive CALPHAD thermodynamic calculations in a Mg-Ca-Zn ternary alloy diffusion simulation. Given alloy composition and temperature, the surrogate predicts thermodynamic and kinetic properties needed by a PDE-based multi-component diffusion solver.

---

## Problem Statement

Diffusion simulations of Mg-Ca-Zn alloys require thermodynamic properties at every spatial node and every time step. Computing these via CALPHAD is accurate but prohibitively slow when called millions of times per simulation. This project trains a fast ML surrogate to replace those calls with negligible loss in accuracy.

The surrogate learns a static mapping:

```
f(T, X_Ca, X_Zn) в†’ (Оә_Ca, Оә_Mg, Оә_Zn, Dv_Ca, Dv_Mg, Dv_Zn, Vm)
```

No temporal architecture is needed вЂ” each prediction is an independent thermodynamic state lookup.

## Dataset

- **75,000 data points** on a perfect full grid: 30 (T) Г— 50 (X_Ca) Г— 50 (X_Zn)
- All inputs are **linearly and uniformly spaced**
- Compositional simplex constraint satisfied throughout: X_Ca + X_Zn + X_Mg = 1
- Data generated from CALPHAD thermodynamic database calculations

## Setup

**1. Clone the repository**
```bash
git clone https://github.com/yourusername/GP1.git
cd GP1
```

**2. Create a virtual environment and install dependencies**
```bash
python -m venv venv
source venv/bin/activate        # on Windows: venv\Scripts\activate
pip install -r requirements.txt
```

**3. Add raw data files**

Place the original CALPHAD output files in `data/raw/`:
```
data/raw/input_data.txt     # columns: T, X_Ca, X_Zn (comma-separated)
data/raw/output_data.txt    # columns: mu_Ca, mu_Mg, mu_Zn, Dv_Ca, Dv_Mg, Dv_Zn, Vm (whitespace-separated)
```

## Preprocessing Design Decisions

**Log-transform diffusion coefficients**
Dv values span ~23 orders of magnitude and follow Arrhenius behaviour. The model predicts `log10(Dv)` and back-transforms to raw scale at inference. Predicting on the raw scale would make the loss function numerically unstable.

**No log-transform on inputs**
All three inputs (T, X_Ca, X_Zn) are linearly and uniformly spaced. StandardScaler is sufficient вЂ” no input log-transform needed.

**Scaler fitted on training data only**
`scaling_parameters.json` stores two sets of scaler parameters: full-dataset (for reference) and train-only (for use in model training). Always use the train-only parameters to avoid data leakage.

**Simplex constraint**
The input space is not a simple hypercube. The constraint `X_Ca + X_Zn в‰¤ 1` (equivalently `X_Mg в‰Ө 0`) is enforced at the data level and validated in `input_bounds_analysis.py`.
